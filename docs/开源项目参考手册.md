# Features Missing from Existing Open Source Projects

The following capabilities are not commonly available in current open-source examination systems and should become the primary innovation points of AES.

## AI-Based Word Paper Parsing

Upload arbitrary Word documents without predefined templates.

Automatically extract:

- Questions
- Options
- Images
- Answers
- Scores
- Knowledge points

Generate structured examination data.

## AI-Assisted Subjective Grading

The AI generates a suggested score and feedback.

The teacher reviews, modifies if necessary, and confirms the final score.

The teacher always retains final authority.

## Java Web Deployment for School LAN

Java Backend

MySQL Database

Optional Python AI Service

Campus LAN Deployment

Browser Access from Computer Labs

## Intelligent Anti-Cheat Logging

Instead of relying on heavy AI vision models, AES should focus on browser-based monitoring:

- Browser focus changes
- Fullscreen state changes
- Tab visibility changes
- Copy / paste attempts
- Page refresh
- Exam interruption logs
- Automatic save and recovery

These events should be recorded as structured audit logs for teacher review.

# Development Principles Derived from Reference Projects

The following principles should guide AES development.

1. Learn architecture, not implementation.
2. Keep business logic independent from frontend.
3. Separate frontend, backend, AI, and database modules.
4. Maintain complete audit logs for all critical operations.
5. Use AI to assist teachers rather than replace their decisions.
6. Prefer modular, replaceable components over tightly coupled designs.
7. Optimize for maintainability and future expansion rather than feature accumulation.
